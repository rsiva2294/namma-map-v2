# SEO Documentation

## Current SEO Implementation

### Metadata Sources

- `index.html` contains the baseline title, description, Open Graph, Twitter, and manifest links.
- `src/App.tsx` updates the document title, description, canonical URL, and social tags dynamically with `react-helmet-async`.
- `src/components/SchemaData.tsx` injects JSON-LD for `WebSite`, `GovernmentService`, and `Dataset`.
- `public/robots.txt` exposes the site and points to the sitemap.
- `public/sitemap.xml` enumerates the base layer and district URLs.

### Heading and Page Structure

- The app is SPA-first and uses a single app shell.
- The primary semantic landmark is `<main id="main-content">`.
- Search, sidebar navigation, and result cards are the main content surfaces.

## Important Findings

1. `src/App.tsx` hardcodes `og:url` to the root URL for every route.
2. `src/App.tsx` builds the canonical URL with `activeLayer.toLowerCase()`, which produces `local_bodies_v2` instead of the sitemap's `local-bodies`.
3. `public/sitemap.xml` and `scripts/generate-sitemap.cjs` generate `local-bodies` URLs, while the live app uses `LOCAL_BODIES_V2` routing.
4. The site is SPA-rendered, so search engines may need more time to fully understand deep pages than they would on pre-rendered pages.

## SEO Improvement Checklist

### High Priority

- Fix the `local-bodies` vs `local_bodies_v2` slug mismatch across router, sidebar, canonical tags, and sitemap.
- Emit route-specific `og:url` values instead of the root URL.
- Consider pre-rendering or SSR for the most important landing pages if organic traffic matters.
- Add a translation-aware canonical strategy for English and Tamil routes if multilingual SEO is a goal.

### Medium Priority

- Add unique, district-specific meta descriptions for the highest-value pages.
- Generate internal links to high-intent landing pages from the homepage or sidebar.
- Add FAQ or how-to content for common searches such as pincode, ration shop, and police jurisdiction lookups.
- Split very large static datasets if first-load performance becomes a crawlability concern.

### Low Priority

- Review branding images for dimensions and file size.
- Add `alt` text and explicit image dimensions where possible.
- Add more descriptive headings inside result cards and landing states.

## Technical SEO Fixes

- Normalize route slugs across `Sidebar`, `RouteManager`, `sitemap.xml`, and canonical tags.
- Make `og:url` and canonical tags route-aware.
- Ensure `index.html` metadata matches the app shell copy.
- Consider static HTML pre-rendering for the most important routes.
- Reduce client-side payload size by lazy-loading or chunking very large district datasets.
